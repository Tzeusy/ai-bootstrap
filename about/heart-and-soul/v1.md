# V1 Boundary

## V1 Ships

- Tool-specific configuration roots for Claude Code, Codex, Gemini CLI, and OpenCode.
- A shared `agents/` reference corpus that preserves older role-specific prompts for selective reuse.
- A shared `skills/` library with reusable prompts, references, assets, and scripts.
- Small repository utilities such as `scripts/refresh.sh` that keep generated or vendored assets reproducible.
- Installation guidance in the main `README.md` for clone-plus-symlink usage.
- Project-shape documents that explain repository doctrine, contracts, topology, and normative requirements.

## V1 Deliberately Defers

- Fully automated cross-tool sync and validation for every mirror surface.
- A formal packaging/release pipeline for publishing skills or agents to third-party registries.
- Centralized machine provisioning or secret-management automation.
- Complete RFC coverage for every individual skill; v1 only establishes the top-level repository contract.

## Platform Targets

- Local developer environments that can symlink or copy repo subdirectories into home-directory tool config paths.
- Multiple agent runtimes with incompatible prompt/config metadata models.

## V1 Success Criteria

- The repository has a clear source-of-truth model.
- New contributors can place a change in the correct layer without guessing.
- Shared assets are distinguishable from tool-specific adapters.
- The repo documents how generated assets are refreshed and where local-only state must not be committed.
