# Heart and Soul

Doctrine for the `ai-bootstrap/` repository.

Read this pillar first when you need to understand what the repository is optimizing for, which operating model it treats as primary, and which boundaries are constitutional rather than incidental.

Reading order:

1. [`vision.md`](./vision.md): thesis, anti-thesis, operating model, and non-negotiable rules.
2. [`v1.md`](./v1.md): what the repository must already do well and what it deliberately defers.
3. [`development.md`](./development.md): contribution and change doctrine for shared assets, mirrors, tracked defaults, and local-only state.
